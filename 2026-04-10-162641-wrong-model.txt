  GLM 模型本身支持结构化输出

  智谱 GLM API 原生支持 response_format={"type": "json_object"} 和 tool
  calling（GLM-4.5/4.6/4.7/5 全系列都支持）。

  问题出在 Vercel AI SDK 的 provider 适配层

  根据 vercel/ai#9002，generateObject 在模型 provider 没有声明 structuredOutputs
   capability 时会失败，抛出 NoObjectGeneratedError — 即使模型实际返回了合法
  JSON。社区版 zhipu-ai-provider 可能没有正确声明这个 capability flag。

  所以真正的原因是：
  - GLM 模型能力上支持 structured output ✅
  - 但你用的 provider 适配器（可能是 OpenAI 兼容模式或社区 zhipu
  provider）没有向 AI SDK 声明 structuredOutputs 支持 ❌
  - AI SDK 的 Output.object() 因此走了 fallback 路径，解析失败

  修复方向

  1. 如果用 OpenAI 兼容模式接入 GLM： 确认 @ai-sdk/openai 的 structMode 设置为
  "tool" 而非 "json"
  2. 如果用社区 zhipu provider： 升级到最新版或检查是否支持 generateObject
  3. 代码层面（Phase 2+ 可做）： 在 Gatekeeper/Grader 中加入 fallback — 当
