## Interaction Contracts

### 1. Page State Machine (D-07)

```
[levels] ---(user clicks "Start Challenge")--> [running]
[running] ---(run_completed event)--> [completed]
[running] ---(run_error event)--> [completed] (with error state)
[running] ---(user clicks Stop)--> [completed] (with cancelled state)
[completed] ---(user clicks "Back to Levels")--> [levels]
```

State type: `{ phase: 'levels' } | { phase: 'running'; runId: string; levelId: string } | { phase: 'completed'; runId: string; levelId: string }`

No URL changes -- internal React state only.

### 2. Level Card Interaction (D-08)

- **Hover:** Card receives `shadow-md` transition (200ms ease), border shifts to `border-primary/20`
- **Click "Start Challenge":** Transitions to RunControls view (intermediate state before run starts)
- **Keyboard:** Card is not a button; only the "Start Challenge" Button inside is focusable

### 3. Run Controls Flow (UI-06)

- **Default state:** Level info displayed (name, description, scenario)
- **Advanced options:** Collapsible panel, collapsed by default, labeled "Advanced Options"
- **Inside advanced:** Three RoleModelSelector rows (Gatekeeper, Challenger, Grader), each showing "Default" as placeholder (inherits global config)
- **Start button:** Disabled while any required field is invalid; primary variant Button

### 4. Conversation Streaming (D-01, D-02, D-05)

- **Gatekeeper message arrives:** Full message appended instantly as left-aligned bubble with Shield icon
- **Challenger delta arrives:** Token appended to streaming bubble on right side; blinking cursor `|` indicator at end
- **Challenger message completes:** Streaming bubble solidifies (cursor removed), next gatekeeper message can arrive
- **Auto-scroll:** `use-stick-to-bottom` wraps conversation; user scroll-up pauses auto-scroll; scroll-to-bottom affordance appears when not at bottom
- **Entry animation:** Each new bubble fades in with `motion/react` `AnimatePresence` (opacity 0->1, translateY 8px->0, duration 200ms)

### 5. Status Bar (D-03)

- **Position:** `sticky top-0 z-10` above conversation stream
- **Content:** `Turn {n}/{maxTurns}` | Status badge (Running/Grading/Completed/Failed) | Token count `{used} tokens`
- **Stop button:** Appears only during `running` status, positioned at right end of status bar, uses `outline` variant Button with `Stop` (Phosphor) icon, 32px height
- **Background:** `bg-background/80 backdrop-blur-sm border-b` for subtle transparency

### 6. Grade Report (D-09, UI-03)

- **Position:** Rendered below the final message in ConversationStream, inside the scroll container
- **Entry animation:** Slides up from below (translateY 16px->0, opacity 0->1, duration 300ms, easeOut)
- **Layout sections (top to bottom):**
  1. **Pass/Fail Banner:** Full-width, `bg-status-success-muted border-status-success-border` for pass, `bg-status-error-muted border-status-error-border` for fail. Display-size heading + Badge
  2. **Required Criteria:** Vertical list, each row: CheckCircle (green) or XCircle (red) icon (16px) + criterion name (font-medium) + reason (text-muted-foreground)
  3. **Performance Dimensions:** Vertical list, each row: Grade badge (A/B/C/D with color per table above) + dimension name + reason
  4. **Improvement Suggestions:** Up to 3 Card components, each containing suggestion text + "Turn {n}" label (text-xs, muted)

### 7. Cancel/Abort (Discretion decision)

- **Supported:** Yes, via AbortController in useArenaSSE
- **UI:** Stop button in ArenaStatusBar (right side), icon-only `Stop` from Phosphor at 16px
- **After cancel:** Status changes to "error" with message "Run cancelled"; partial conversation preserved in view; grade report does not appear
- **No confirmation dialog:** Cancel is instant (non-destructive -- conversation is already persisted server-side per DATA-04)

## Accessibility Contracts

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | All interactive elements (buttons, selects, collapsibles) use Radix primitives with built-in keyboard support |
| Focus management | After "Start Challenge" click, focus moves to status bar stop button; after run completes, focus moves to grade report heading |
| Screen reader: role labels | Gatekeeper/Challenger role name announced via `aria-label` on each bubble container |
| Screen reader: live region | Status bar uses `aria-live="polite"` for status changes; grade report banner uses `aria-live="assertive"` |
| Screen reader: streaming | Streaming bubble has `aria-busy="true"` while streaming; final content announced on completion |
| Color contrast | All text meets WCAG AA (4.5:1 minimum). oklch tokens in globals.css are pre-validated for stone palette. Blue-500 on bg-blue-500/10 passes AA for large text. |
| Reduced motion | Wrap all `motion` animations in `prefers-reduced-motion` media query via motion/react's `useReducedMotion` |
