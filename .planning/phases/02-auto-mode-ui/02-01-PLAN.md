2. 创建 `src/hooks/useArenaSSE.ts`，实现 Arena SSE 消费 hook：

- 导入 `authFetch` from `@/lib/api-client`
- 导入 `ArenaUIMessage`, `ArenaRunUIStatus`, `RunParams`, `UseArenaSSEReturn` from `@/components/arena/types`
- 导入 `GraderOutput` from `@/arena/schemas/grader-output`
- 导入 `ArenaSSEEvent` from `@/arena/types`（仅用于类型）

关键实现逻辑：
- `startRun(params)`: 重置所有状态 -> 创建 AbortController -> authFetch POST `/api/arena/run`，body 为 `{ worldId, levelId, providerId: params.gatekeeperProviderId, model: params.gatekeeperModel }` -> 获取 ReadableStream reader -> 逐行解析 `data: ` 前缀行 -> JSON.parse 一次得到 `{ type, data }` -> 调用 handleEvent 分发
- `cancelRun()`: 调用 abortRef.current?.abort() -> 设置 status='error', error='Run cancelled'
- 组件卸载时自动 abort（useEffect cleanup）

handleEvent 分发逻辑（基于 event.type）：
- `run_started`: setRunId(data.runId), setStatus('running')
- `gatekeeper_message`: setMessages(prev => [...prev, { id: crypto.randomUUID(), role: 'gatekeeper', content: data.content, turn: data.turn }])
- `challenger_delta`: setStreamingDelta(prev => prev + data.delta) -- 追加而非赋值
- `challenger_message`: 将完整 Challenger 消息固化到 messages[], 清空 streamingDelta
- `turn_completed`: setCurrentTurn(data.turn)
- `grading_started`: setStatus('grading')
- `grade_result`: setGrade(data as GraderOutput)
- `run_completed`: setStatus('completed')
- `run_error`: setStatus('error'), setError(data.error)
- `token_usage`: setTokenUsage({ totalUsed: data.totalUsed, remaining: data.remaining })

SSE 解析注意事项（避免 Pitfall 1）：
- Arena SSE 的 `data:` 行格式为完整 JSON `{"type":"...", "data":{...}}`
- 只需外层 JSON.parse 一次，内部 data 字段已经是对象
- 使用 TextDecoder with `{ stream: true }` 处理分段传输
- buffer 拼接逻辑：将未完成的行留在 buffer 中

AbortController 清理（避免 Pitfall 6）：
- useRef 存储 AbortController
- useEffect 返回 cleanup 函数调用 abort
- startRun 开始时如果有旧的 controller 先 abort

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | useArenaSSE | mitigate | 使用 authFetch 自动附加 Bearer token，后端 requireAuth 验证（Phase 1 已实现） |
| T-02-02 | Tampering | SSE event data | accept | SSE 数据从服务端生成，客户端只读消费，不存在篡改风险 |

| T-02-04 | Denial of Service | AbortController | mitigate | 组件卸载时自动 abort SSE 连接，防止资源泄漏；cancelRun 提供手动终止能力 |
